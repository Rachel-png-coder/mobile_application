# 🏙️ Kigali City Services & Places Directory

A comprehensive Flutter mobile application that helps Kigali residents locate and navigate to essential public services, leisure spots, and tourist attractions.

##  Features

### 1. User Authentication
- **Email/Password Sign Up**: Create a new account with email verification
- **Secure Login**: Authenticated access to the app
- **Email Verification**: Users must verify their email before accessing the app
- **User Profiles**: Each user has a profile stored in Firestore
- **Secure Logout**: Proper session management

### 2. Location Listings (CRUD Operations)

#### Create
- Add new service locations with complete details
- Choose from 8 categories: Hospital, Police Station, Library, Restaurant, Café, Park, Tourist Attraction, Utility Office
- Enter address, contact number, and description
- Set geographic coordinates (latitude/longitude)

#### Read
- Browse all listings in real-time
- View detailed information about each location
- See listings organized chronologically

#### Update
- Edit listings created by the authenticated user
- Modify any field of your own listings

#### Delete
- Remove listings created by the authenticated user
- Confirmation dialog prevents accidental deletion

### 3. Search and Filter
- **Search by Name**: Real-time search as you type
- **Filter by Category**: Click category chips to filter listings
- **Dynamic Updates**: Results update instantly as you search/filter

### 4. Map Integration
- **Embedded Google Maps**: View location on map in detail page
- **Location Marker**: Pinpoint exact location with marker
- **Directions**: Launch turn-by-turn navigation in Google Maps

### 5. User Interface
- **Bottom Navigation**: Four main screens for easy access
  - 📋 **Directory**: Browse all listings
  - 📍 **My Listings**: Manage your own listings
  - 🗺️ **Map View**: All listings on interactive map
  - ⚙️ **Settings**: Profile and preferences

### 6. Settings
- **Profile Information**: View your email and account details
- **Notification Toggle**: Enable/disable location-based notifications (simulated)
- **Debug Info**: View your user ID for troubleshooting

### 7. Real-time Updates
- All changes reflect immediately across all screens
- No manual refresh needed - powered by Firestore streams

## 🗄️ Firestore Database Structure

### Collections Overview
Firestore Root
├── users/ # User profiles
└── listings/ # Service locations


### 1. Users Collection

**Collection ID:** `users`
**Document ID:** User UID (matches Firebase Auth UID)

```javascript
// Example user document
{
  uid: "user_123456",                    // Document ID (matches Auth UID)
  email: "emailhere",          // User's email address
  displayName: "yourname",                 // Optional display name
  phoneNumber: "+250788123456",            // Optional contact number
  createdAt: March 9, 2025 at 10:30:00 AM UTC+2,  // Account creation date
  notificationsEnabled: true                // User preference for notifications
}

2. Listings Collection
Collection ID: listings
Document ID: Auto-generated by Firestore

// Example listing document
{
  id: "abc123xyz789",                      // Auto-generated document ID
  name: "Kigali Central Hospital",          // Place/service name
  category: "Hospital",                     // One of 8 predefined categories
  address: "KG 123 St, Kigali",             // Physical address
  contactNumber: "+250788123456",            // Phone contact
  description: "24/7 emergency services",    // Detailed description
  latitude: -1.9441,                         // Geographic coordinate
  longitude: 30.0619,                         // Geographic coordinate
  createdBy: "user_123456",                   // References users.uid
  timestamp: March 9, 2025 at 10:35:00 AM UTC+2,  // Creation/modification time
  imageUrl: "https://example.com/image.jpg"   // Optional listing image
}

Purpose: Stores all service and location information with ownership tracking.

Indexes Created
To optimize query performance, the following composite indexes were created:

Collection	Fields	Purpose
listings	createdBy (Ascending), timestamp (Descending)	Fetch user's listings in chronological order
listings	category (Ascending), timestamp (Descending)	Fast category-based filtering

Firestore Security Rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users collection - users can read/write only their own profile
    match /users/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && request.auth.uid == userId;
    }
    
    // Listings collection - authenticated users can read all,
    // but only owners can modify their own
    match /listings/{listing} {
      allow read: if request.auth != null;
      allow create: if request.auth != null 
                    && request.resource.data.createdBy == request.auth.uid;
      allow update, delete: if request.auth != null 
                           && resource.data.createdBy == request.auth.uid;
    }
  }
}

 #1. AuthProvider
#Purpose: Manages authentication state and user profile

Key Responsibilities:

Listens to Firebase Auth state changes

Handles sign up, login, and logout flows

Tracks email verification status

Fetches and caches user profile data

Provides user information to other parts of the app

The Kigali City Services Directory uses Provider as its state management solution. Provider was chosen for its simplicity, excellent integration with Flutter, and seamless compatibility with Firebase streams.




###State Management: Provider
The application uses the Provider package for state management with the following providers:
1. AuthProvider
Manages user authentication state
Handles sign up, login, logout
Tracks email verification status
Stores user profile data
2. ListingProvider
Manages all listing data
Real-time Firestore streams
Search and filter logic
CRUD operations

3. SettingsProvider
